# Caller AI

Система для автоматического анализа звонков отдела продаж, контроля качества и генерации обратной связи для менеджеров. Основной упор в проекте сделан на агентный LLM-пайплайн на базе GigaChat с использованием RAG.

## Архитектура

Проект состоит из трех основных частей:
- Frontend: React и Vite в папке /frontend для просмотра транскриптов и аналитических отчетов.
- Backend: FastAPI в папке /app для API, работы с базой данных и загрузки данных.
- Агентная LLM-система: находится в /app/llm_module и отвечает за парсинг диалогов, поиск нарушений и формирование рекомендаций.

## Агентная система и RAG

Основная логика анализатора реализована в виде мультиагентного конвейера. Каждый агент решает узкую задачу и передает контекст дальше. Пайплайн запускается через модуль agents.py и проходит следующие этапы:

Сначала агент-структуризатор разбивает сырой транскрипт на этапы диалога, используя текстовые знания о структуре звонка. Затем агент контроля качества проверяет соответствие разговора эталонному скрипту продаж и ищет отклонения. Следом подключается агент-аудитор, который выявляет грубые ошибки менеджера по заранее заданным критериям. На основе этих ошибок агент-коуч формирует персональные рекомендации, используя лучшие практики. В конце агент-агрегатор объединяет все выводы в единый JSON-отчет.

Интеграция с RAG работает через векторную базу ChromaDB (/chroma_db/ и модуль rag.py). Она позволяет динамически подтягивать в промпты актуальные документы, примеры диалогов и правила в зависимости от контекста текущего звонка.

Конфигурация логики агентов и текстовые сценарии находятся в app/llm_module/knowledge: там лежат этапы разговора (stages.txt), эталонный скрипт (script.txt), критерии оценки (criteria.txt) и шаблоны рекомендаций (coach_tips.txt).

## Запуск проекта

Для авторизации в API потребуется ключ GigaChat. В корне проекта нужно создать файл .env со следующими параметрами:

```env
GIGACHAT_AUTH_KEY=ваш_ключ_авторизации
GIGACHAT_SCOPE=GIGACHAT_API_PERS
GIGACHAT_MODEL=GigaChat-2-Pro
GIGACHAT_AUTH_URL=https://ngw.devices.sberbank.ru:9443/api/v2/oauth
GIGACHAT_API_URL=https://gigachat.devices.sberbank.ru/api/v1/chat/completions
```

Обратите внимание, что в GIGACHAT_AUTH_KEY передается Authorization Key из панели разработчика, а не токен. Получение access-токена система делает сама через gigachat_client.py.

Для запуска бэкенда и LLM:
```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --reload
```

Вы также можете проверить работу системы изолированно от сервера базы данных, запустив скрипт python3 app/llm_module/test_agents.py. Он отдаст результат анализа в консоль и запишет выводы в текстовый файл отчета.

Для запуска веб-интерфейса:
```bash
cd frontend
npm install
npm run dev
```

## Данные и результаты

LLM-система возвращает данные на бэкенд в структурированном виде. Ответ представляет собой JSON с итоговой оценкой качества звонка, массивом выявленных ошибок, советами по улучшению, выводами по этапам сделки и статистикой потраченных токенов.

В ближайших планах проекта — оптимизация пайплайна для сокращения потребления токенов, перевод агентов на параллельную асинхронную работу и улучшение парсинга документов для пополнения векторной базы.
